Project Overview — Aidan McCollick (Cinematic Narrative Photography Portfolio)

Project Name

8CigsN4Roses — Cinematic Narrative Photography Portfolio

Project Overview

Function / Purpose

A cinematic, story-driven portfolio site for a film photographer that highlights creative direction, editing, and styling through interactive, scroll-based “photo stories.” The site improves visibility, invites collaboration, and communicates process.

Intended Users

  • Art directors, stylists, brands, editorial teams (potential clients)
  • Followers of film-based, narrative photography
  • Creative collaborators (MUA, wardrobe, set design, assistants)

Overview of the Content

  • Cinematic homepage reel and featured projects
  • Project pages with concept, inspiration, BTS, edit notes, and final images
  • Interactive “Before/After” grading sliders
  • Interactive “Build a Moodboard” tool (share/download)
  • About, Contact, and passworded Client Proofs

Page Design

Home (index.html)

  • Purpose: Set cinematic tone; route to key stories.
  • Audience: All visitors.
  • Content: Full-bleed reel or cross-fade hero; featured projects; brief thesis line.
  • Data entry: No.
  • Validation: N/A.
  • Links/Controls: Global nav; “View Story” buttons.
  • Actions: Smooth scroll; lazy-load images; play/pause hero video.
  • Notes: Subtle film-grain overlay; accessible contrast.

Projects (projects.html)

  • Purpose: Browse all projects.
  • Audience: Clients, fans.
  • Content: Grid/list with thumbnails, loglines, tags (portrait, editorial, monochrome).
  • Data entry: No.
  • Validation: N/A.
  • Links/Controls: Filters (tag buttons), sort dropdown.
  • Actions: Filter/sort via JS; deep-link to project pages.
  • Notes: jQuery-UI Accordion or Tabs acceptable here.

Project Story (project-slug.html)

  • Purpose: Scroll-driven narrative (concept → inspo → BTS → edit notes → finals).
  • Audience: Clients, collaborators.
  • Content: Images, captions, brief text sections, short clips.
  • Data entry: Optional comment form (read-only handling only).
  • Validation: If form present, validate required fields and email format (client-side).
  • Links/Controls: “Before/After” slider; next/previous project; “Add to Moodboard.”
  • Actions: Scroll animations; update URL hash by chapter; slider drag events.
  • Notes: Avoid server-side storage; use localStorage for preferences.

Moodboard Builder (moodboard.html)

  • Purpose: Let users curate a shareable/downloadable collage of selected images.
  • Audience: Visitors, art directors.
  • Content: Draggable thumbnails, canvas export preview.
  • Data entry: No account; local only.
  • Validation: N/A.
  • Links/Controls: “Add/Remove,” “Clear,” “Download PNG,” “Copy Share Link.”
  • Actions: Use Canvas API for export; optional querystring to reconstruct a board.
  • Notes: No server storage; comply with read-only rule.

About (about.html)

  • Purpose: Communicate persona and values.
  • Audience: All visitors.
  • Content: Short bio, portrait, ethos statement.
  • Data entry: No.
  • Validation: N/A.
  • Links/Controls: Social links; “Contact.”
  • Actions: None beyond navigation.
  • Notes: Keep mysterious/minimal; consider handwritten caption style.

Contact / Client Proofs (contact.html / proofs.html)

  • Purpose: Inquiries and passworded client galleries (read-only proofs).
  • Audience: Clients.
  • Content: Contact form; proof thumbnails; approve/reject toggles (client-side only).
  • Data entry: Yes—contact fields; proof selection.
  • Validation: Required + email pattern; length checks.
  • Links/Controls: Submit; filter; approve/reject (local state or existing API).
  • Actions: AJAX form to email/third-party without reload; update UI state for proofs.

Dynamic Functionality

Description & Purpose

  • Before/After Slider (Project Story): Show RAW vs final grade; communicates editing skill.
  • Moodboard Builder (Moodboard): Drag-to-collect; export to PNG; optional share link.
  • Scroll Animations (Home/Project Story): Cinematic reveals; chapter markers; deeper engagement.
  • AJAX Contact (Contact): Send form to email/third-party without page reload.
  • jQuery-UI Widget (Projects): Tabs/Accordion for filtering or FAQ; accessible interactions.

Requirements Checklist

  • 5+ pages with single-click nav
  • Consistent header/footer (footer includes HTML/CSS validation links)
  • JavaScript interactivity (sliders, filters, scroll effects)
  • jQuery-UI widget (Tabs/Accordion)
  • AJAX form submission (Contact)
  • Wireframe and Site Map embedded with links to Draw.io
  • Technical writing; no grammar/spelling errors
